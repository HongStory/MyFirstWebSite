OS part2

    CPU 스케줄링 개요

  • 운영체제가 프로세스들에게 공정하고
    합리적으로 CPU자원을 배분하는것
  • 프로세스 우선순위
  • 우선순위는 PCB에 저장
  • 스케줄링 큐
  • CPU쪽에 줄서기
  • 하드디스크쪽에 줄서기
  • 프린터쪽에 줄서기
  • 준비큐 언제든지 실행될 준비됨
  • 대기큐 입출력요청와서 잠시 대기중

    • 선점형과 비선점형 스케줄링

  • 선점형:새치기 가능
  • 자원 독점을 막는다
  • 문맥교환자주발생 오버헤드초래
  • 비선점형:끝날때까지 기다림

    • CPU스케줄링 알고리즘

  • 선입 선처리(FCFS)
  • First Come First Served
  • 단순히 준비 큐에 삽입된 순서대로 처리
  • 다음 프로세스의 대기 시간이 길어진다
  • 최단 작업 우선(SJF)
  • Shortest Job First
  • 호위 효과를 방지 하려면
  • CPU사용 시간이 긴 프로세스는 나중에
  • CPU사용 시간이 짧은 프로세스는 먼저
  • 라운드 로빈(RR)
  • Round Robin
  • 선입 선처리 스케줄링+타임슬라이스
  • 각 프로세스가 CPU를
    사용할 수 있는 정해진 시간
  • 정해진 타임 슬라이스만큼의 시간
    동안 돌아가며 CPU를 이용하는
    선점형 스케줄링
  • 최소 잔여 시간 우선
  • Shortest Remaining Time
  • 최단 작업 우선 + 라운드 로빈
  • 정해진 시간만큼 CPU를 이용하되
    다음으로 CPU를 사용할 프로세스는
    남은 작업 시간이 가장 적은 프로세스
  • 우선 순위
  • 우선순위를 부여하고 높은 프로세스 순
  • 우선순위가 같으면 선입 선처리
  • 기아 현상:우선순위 높은 프로세스만
    주구장창 실행,낮은 프로세스는 계속
    실행 연기됨
  • 이를 방지하기 위해 에이징 기법사용
  • 다단계 큐
  • Multilevel queue
  • 우선 순위 스케줄링의 발전된 형태
  • 우선 순위별로 준비 큐를 여러 개 사용
  • 큐가 비면 다음 큐로 이동
  • 큐간 이동이 불가 역시 기아현상 발생
  • 다단계 피브백 큐
  • Multilevel feedback queue
  • 다단계 큐 스케줄링의 발전된 형태
  • 큐간의 이동가능
  • 에이징 기법 적용가능

    • 프로세스 동기화

  • 동시다발적으로 실행되는 프로세스들은
    서로 협력하며 영향을 주고 받는다
    이 과정에서 자원의
    일관성을 보장해야함
  • 공동의 목적을 위해 동시에
    수행되는 프로세스
  • 프로세스들의 수행 시기를 맞추는 것
  • 실행 순서 제어:프로세스들을
    올바른 순서대로 실행하기
  • 상호 배제:동시에 접근해서는 안되는
    자원에 하나의 프로세스만
    접근하게 하기
  • 실행 순서 예시
  • writer reader problem
  • 쓰기가 먼저되야 읽을 수 있다
  • 상호 배제 예시
  • Bank account problem
  • 현재잔액에 2만원과 5만원을 더하기
  • Producer,Consumer problem
  • 동시에 접근해서는 안되는 자원
  • 공유자원과 임계 구역
  • 전역 변수,파일,
    비출력장치,보조기억장치
  • 총합변수,잔액변수
  • 임계 구역에 동시에 접근하면
    자원의 일관성이 깨질 수 있다
  • 이를 레이스 컨디션(race condition)
  • 임계구역 문제를 해결하는 세가지 원칙
  • (상호 배제를 위한 동기화를 위한 )
  • 상호배제:한 프로세스가 임계 구역에
    진입했다면 다른 프로세스는 못 들어옴
  • 진행:임계구역에 어떤 프로세스도 진입
    하지 않았다면 진입하고자 하는
    프로세스는 들어갈 수 있다
  • 유한 대기:한 프로세스가 임계 구역에
    진입하고 싶다면 언젠가는 임계 구역에
    들어올 수 있어야한다
    (무한정 대기해서는 안된다)

    • 동기화 기법

  • 뮤텍스 락
  • 상호 배재를 위한 동기화 도구
  • 프로세스=>탈의실(임계구역)
  • 뮤텍스 락의 단순한 형태
  • 전역변수하나,함수두개
  • 자물쇠 역할:전역변수
  • 잠그는 역할:함수1
  • 해체하는 역할:함수2
  • 바쁜대기(busy waiting)
  • 세마포
  • 좀더 일반화된 방식의 동기화 도구
  • 공유자원이 여러개 있는
    경우도 적용가능
  • 임계구역 앞에서 멈춤신호를
    받으면 잠시대기
  • 임계구역 앞에서 가도 좋다는
    신호를 받으면 임계구역 진입
  • 세마포의 단순한 형태
  • 임계구역에 진입할 수 있는
    프로세스의 개수를 나타내는
    전역 변수S
  • 임계구역에 들어가도 좋은지,
    기다려야 할지를 알려주는 wait함수
  • 임계구역 앞에서 기다리는 프로세스에
    가도 좋다고 신호를 주는 signal함수
  • 모니터
  • 사용자(개발자)가 사용하기 편한
    동기화 도구
  • 상호배제를 위한 동기화
  • 인터페이스를 위한 큐
  • 공유자원에 접근하고자 하는
    프로세스를 큐에 삽입
  • 큐에 삽입된 순서대로 공유 자원 이용
  • 실행 순서 제어를 위한 동기화
  • 조건 변수condition variable이용
  • 프로세스나 스레드의 실행순서를
    제어하기 위해 사용하는 특별한 변수
  • 특정 프로세스가 아직 실행될 조건
    이 되지 않았을 때에는 wait을
    통해 실행 중단
  • 특정 프로세스가 실행될 조건이
    충족되었을 때에는 signal을
    통해 실행 재개

    • 교착 상태란

  • 서로가 가진 자원을 서로 기다리는중
  • 해결하기 위해서는
  • 발생했을때의 상황을 정확히 표현해보기
  • 근본적인 이유 이해하기
  • 자원 할당 그래프
  • 프로세스는 원
  • 자원은 사각형
  • 자원의 갯수는 사각형 안에 점
  • 할당 받았다면 점에서 원으로 화살표
  • 자원 기다리고 있다면 사각형으로 화살표
  • 교착상태가 발생하는 조건
  • 상호배재:한 프로세스가 사용하는
    자원을 다른 프로세스가
    사용할 수 없는 상태
  • 점유와 대기:자원을 할당 받은
    상태에서 다른 자원을 기다리는 상태
  • 비선점:어떤 프로세스도 다른 프로세스
    의 자원을 강제로 빼앗지 못하는 상태
  • 원형 대기:프로세스들이 원의
    형태로 자원을 대기하는 상태
  • 4가지 조건을 모두 충족해야
    교착상태 발생
  • 교착상태 해결 방법
  • 예방
  • 애초에 발생하지 않도록
  • 발생 조건중 하나를 없애버리기
  • 회피
  • 무분별한 자원 할당으로
    인해 발생했다고 간주
  • 발생 하지 않을 만큼 조심조심 할당
  • 자원의 양을 고려하여 발생하지
    않을 만큼만 배분
  • 안전 순서열:안전하게 프로세스들에
    자원을 할당할 수 있는 순서
  • 안전 상태:모든 프로세스가 자원을
    할당 받고 종료될 수 있는 상태
  • 불안전 상태:교착 상태가 발생할
    수도 있는 상태
  • 검출 후 회복
  • 사후 조치하는 방식
  • 일단 할당,검출 되면 회복
  • 선점을 통한 회복:해결 될때까지
    한 프로세스씩 자원을 몰아주는 방식
  • 강제 종료를 통한 회복:
    작업 내역을 잃을 위험
  • 한 프로세스씩 강제 종료(오버헤드)
  • 교착상태무시
